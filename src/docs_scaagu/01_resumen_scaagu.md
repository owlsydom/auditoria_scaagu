# Resumen del Caso Banco Andes Capital

Banco Andes Capital es una entidad ficticia dedicada a servicios financieros. El objetivo de esta auditoría es evaluar vulnerabilidades en su portal de clientes, utilizando DVWA (Damn Vulnerable Web Application) en un entorno controlado.

## Objetivo de la Auditoría

Se realizó una evaluación integral de seguridad enfocada en identificar y demostrar vulnerabilidades críticas en la aplicación web. Se demostraron tres ataques principales:

- **SQL Injection** - Acceso no autorizado a bases de datos
- **Cross-Site Scripting (XSS)** - Ejecución de código malicioso en navegadores
- **Command Injection** - Ejecución de comandos arbitrarios en el servidor

## Metodología

Se utilizó una metodología de pruebas de penetración estructurada, midiendo la severidad de cada vulnerabilidad con el estándar CVSS (Common Vulnerability Scoring System) y proponiendo controles de seguridad específicos para cada caso.

## Hallazgos Principales

Se identificaron **3 vulnerabilidades críticas** que requieren remediación inmediata, junto con recomendaciones de controles de seguridad específicos para mitigar el riesgo.

---

**Conclusión:** El sistema requiere mitigación inmediata de vulnerabilidades críticas antes de desplegar a producción.
